Routes

In Laravel, all routes are defined in the routes/web.php file for web interfaces or routes/api.php for API routes.

Router Methods


Route::get($uri, $callback);
Route::post($uri, $callback);
Route::put($uri, $callback);
Route::patch($uri, $callback);
Route::delete($uri, $callback);
Route::options($uri, $callback);
Route::view($uri, $view, $data = []);
Route::redirect($from, $to, $status = 302);

uri: '/' or '/some/route'

callback: function () { // ... } or [Controller::class, 'myfunction']

Route Parameters


Route::get('/user/{id}', function ($id) {
    return 'User ID: ' . $id;
});

Named Routes

You can define named routes to easily reference them in your application:


Route::get('/profile', [UserController::class, 'show'])->name('profile.show');

Refer to a named route using:


route('profile.show');

Views

In Laravel, views are located in the resources/views directory.

Returning Views


return view('welcome'); // resources/views/welcome.blade.php
return view('auth.login'); // resources/views/auth/login.blade.php

Passing Data to Views


return view('profile', ['name' => 'Michaelninder']); // fixed value
return view('profile', ['name' => $name]); // variable
return view('profile', compact('name')); // if both values (variable and datavalue have the same name)

Blade Directives


@if ($user)
    Hello, {{ $user->name }}
@endif

Laravel Blade also supports layouts and components:

Controllers

Controllers are responsible for handling HTTP requests and returning responses.

Basic Controller Example


class AuthController extends Controller
{
    public function showLogin()
    {
        return view('auth.login');
    }
}

Defining Route to Controller


Route::get('/login', [AuthController::class, 'showLogin']);

Controller with Dependency Injection


class UserController extends Controller
{
    public function showProfile(User $user)
    {
        return view('profile', compact('user'));
    }
}

Resource Controllers

You can generate a resource controller to handle CRUD operations:


php artisan make:controller PostController --resource

Then, you can define the routes to resource methods in your routes file:


Route::resource('posts', PostController::class);

Middleware

Middleware provides a convenient mechanism for filtering HTTP requests entering your application.

Creating Middleware


php artisan make:middleware CheckAge

Defining Middleware Logic


class CheckAge
{
    public function handle($request, Closure $next)
    {
        if ($request->age < 18) {
            return redirect('home');
        }

        return $next($request);
    }
}

Applying Middleware


Route::get('/profile', [ProfileController::class, 'index'])->middleware('check.age');

Validation

Laravel provides powerful validation functionality to validate incoming data.

Basic Validation


$request->validate([
    'email' => 'required|email',
    'password' => 'required|min:6',
]);

Custom Error Messages


$request->validate([
    'email' => 'required|email',
], [
    'email.required' => 'We need to know your e-mail address!',
]);

Authentication

Laravel offers a built-in authentication system for handling login and registration.

Authentication Scaffolding

Use the following Artisan command to scaffold authentication views and routes:


php artisan make:auth

Protecting Routes with Auth


Route::get('/dashboard', [DashboardController::class, 'index'])->middleware('auth');

Logging Out


public function logout()
{
    Auth::logout();
    return redirect('/login');
}

Database & Eloquent

Laravel's Eloquent ORM makes it easy to interact with your database.

Database Migrations

Create a migration using Artisan:


php artisan make:migration create_posts_table

Eloquent Models

Eloquent provides an easy way to interact with your database records:


class Post extends Model
{
    protected $fillable = ['title', 'content'];
}

Querying with Eloquent


$posts = Post::where('status', 'published')->get();

Inserting Data


Post::create([
    'title' => 'New Post',
    'content' => 'Post content here...',
]);

Advanced Features

Laravel provides several advanced features for building powerful applications:

  • Queues: Handle background tasks asynchronously.
  • Events & Listeners: Listen for and handle events in your application.
  • Jobs: Encapsulate logic to be run asynchronously or in the background.
  • Notifications: Send notifications via various channels like email, SMS, and database.
  • Mail: Send email from your application.
  • Task Scheduling: Automatically schedule tasks such as clearing logs or sending daily emails.
