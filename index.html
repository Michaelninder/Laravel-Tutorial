Laravel Tutorial

Routes

In Laravel, all routes are defined in the routes/web.php file for web interfaces or routes/api.php for API routes.

Router Methods


Route::get($uri, $callback);
Route::post($uri, $callback);
Route::put($uri, $callback);
Route::patch($uri, $callback);
Route::delete($uri, $callback);
Route::options($uri, $callback);
Route::view($uri, $view, $data = []);
Route::redirect($from, $to, $status = 302);

uri: '/' or '/some/route'

callback: function () { // ... } or [Controller::class, 'myfunction']

Route Parameters


Route::get('/user/{id}', function ($id) {
    return 'User ID: ' . $id;
});

Named Routes

You can define named routes to easily reference them in your application:


Route::get('/profile', [UserController::class, 'show'])->name('profile.show');

Refer to a named route using:


route('profile.show');

Views

In Laravel, views are located in the resources/views directory.

Returning Views


return view('welcome'); // resources/views/welcome.blade.php
return view('auth.login'); // resources/views/auth/login.blade.php

Passing Data to Views


return view('profile', ['name' => 'Michaelninder']); // fixed value
return view('profile', ['name' => $name]); // variable
return view('profile', compact('name')); // if both values (variable and datavalue have the same name)

Blade Directives


@if ($user)
    Hello, {{ $user->name }}
@endif

Layouts & Components

Laravel Blade also supports layouts and components:


<!-- resources/views/layouts/app.blade.php -->
<html>
  <head>
    <title>@yield('title')</title>
  </head>
  <body>
    @yield('content')
  </body>
</html>
          

<!-- resources/views/home.blade.php -->
@extends('layouts.app')

@section('title', 'Home Page')

  <h1>Welcome to Home!</h1>
@endsection

Controllers

Controllers are responsible for handling HTTP requests and returning responses.

Basic Controller Example


class AuthController extends Controller
{
    public function showLogin()
    {
        return view('auth.login');
    }
}

Defining Route to Controller


Route::get('/login', [AuthController::class, 'showLogin']);

Controller with Dependency Injection


class UserController extends Controller
{
    public function showProfile(User $user)
    {
        return view('profile', compact('user'));
    }
}

Resource Controllers

You can generate a resource controller to handle CRUD operations:


php artisan make:controller PostController --resource

Then, you can define the routes to resource methods in your routes file:


Route::resource('posts', PostController::class);

Middleware

Middleware provides a convenient mechanism for filtering HTTP requests entering your application.

Creating Middleware


php artisan make:middleware CheckAge

Defining Middleware Logic


class CheckAge
{
    public function handle($request, Closure $next)
    {
        if ($request->age < 18) {
            return redirect('home');
        }

        return $next($request);
    }
}

Applying Middleware


Route::get('/profile', [ProfileController::class, 'index'])->middleware('check.age');

Validation

Laravel provides powerful validation functionality to validate incoming data.

Basic Validation


$request->validate([
    'email' => 'required|email',
    'password' => 'required|min:6',
]);

Custom Error Messages


$request->validate([
    'email' => 'required|email',
], [
    'email.required' => 'We need to know your e-mail address!',
]);

Localization

Laravel makes it easy to manage application translations and localize your content for different languages and regions.

Creating Language Files

Language files are stored in the resources/lang directory. You can create language-specific files within this directory (e.g., resources/lang/en/messages.php).


return [
    'welcome' => 'Welcome to our application!',
];

Using Localization in Views

To display translated text, use the @lang directive:


<h1>@lang('messages.welcome')</h1>

Setting the Locale

You can set the applicationâ€™s locale in config/app.php:


'locale' => 'en',

Or dynamically change it using the App::setLocale() method:


App::setLocale('fr');

Pluralization

Laravel can handle plural forms with the @choice directive:


@choice('messages.apples', 5)

In the language file, you can define the pluralization rules:


return [
    'apples' => '{1} :count apple|[2,*] :count apples',
];

Blade Advanced

Laravel Blade is a powerful templating engine. It provides features like loops, conditionals, and includes for dynamic and reusable views.

Blade Loops

You can use loops in Blade to iterate over collections or arrays:


@foreach ($users as $user)
    <p>{{ $user->name }}</p>
@endforeach

Blade Conditionals

Use @if, @else, and @elseif for conditional statements:


@if ($user->isAdmin())
    <p>Welcome, admin!</p>
@else
    <p>Welcome, regular user!</p>
@endif

Including Views

You can include Blade views within other views using the @include directive:


@include('partials.header')

Blade Components

Components allow you to create reusable pieces of HTML:


<x-alert type="success" :message="$message" />

Component view (e.g., resources/views/components/alert.blade.php):


<div class="alert alert-{{ $type }}">
    {{ $message }}
</div>

Blade Layouts

Use layouts to structure common HTML across multiple views:


<!-- resources/views/layouts/app.blade.php -->
<html>
  <body>
    @yield('content')
  </body>
</html>
	      

<!-- resources/views/home.blade.php -->
@extends('layouts.app')

@section('content')
  <h1>Home Page</h1>
@endsection

API Resources

Laravel provides API resources to transform models and collections into JSON easily.

Creating a Resource


	  php artisan make:resource UserResource

Example Resource Class


	  class UserResource extends JsonResource
	  {
	      public function toArray($request)
	      {
	          return [
	              'id' => $this->id,
	              'name' => $this->name,
	              'email' => $this->email,
	          ];
	      }
	  }

Using a Resource


	  return new UserResource($user);
